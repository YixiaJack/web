The Four Moving Parts

A JEPA looks more complicated than a standard autoencoder, but the building blocks are familiar. There is a context encoder that produces a representation of what is visible, a target encoder that produces a representation of what is hidden, a predictor that maps from one to the other in representation space, and an energy / loss term that pulls the prediction toward the truth. What Makes JEPA Non-Trivial

The architecture has a well-known failure mode: representation collapse. If the encoder is allowed to map every input to the same constant vector, the predictor can match the target trivially — perfect loss, useless features. JEPA prevents this with two design choices working together:

  1. Asymmetry between encoders. The target encoder is an exponential-moving-average copy of the context encoder, with no gradient flowing back through it. The student has to chase a slowly-moving teacher rather than negotiate a degenerate fixed point.
  2. Block-level masking. I-JEPA samples large rectangular target blocks rather than tiny patches. Predicting a large block forces the encoder to capture object-level structure; if it collapsed, even the simplest block would be unpredictable.

These two ingredients are also what differentiate JEPA from contrastive methods like SimCLR or MoCo, which prevent collapse by hard-negative sampling. JEPA does not need negatives at all.

Notation

  • x — context input (visible part)
  • y — target input (hidden part)
  • s_x — context representation
  • s_y — target representation
  • ŝ_y — predicted target representation
  • z — latent that conditions the predictor
  • EMA — exponential moving average

Why an EMA Teacher

Stopping the gradient and using an EMA copy for the target encoder is a trick borrowed from BYOL. It produces stable targets that drift slowly enough for the student to chase, but not so slowly that learning stalls. The EMA momentum is a small but consequential hyperparameter.
